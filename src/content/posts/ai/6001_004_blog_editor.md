# 流程

## 1、想法

这博客不是一次性完成的，需要想每一处怎么实现，AI给你定个整体风格就可以了，其他细节交互细节最好自己实现。

## 2、原型图

很多大佬都是自己去[Figma](https://www.figma.com/)、[Google AI Studio](https://aistudio.google.com/)、[Google Stitch](https://stitch.withgoogle.com/)这些专业的AI生成原型图网站

这个办法没问题，而且很真确。但是你会发现，跟网页端AI对线后，还要跟本地IDE进行对线。本地的IDE还不一定根据你原型图进行。。。费时又费力，怎么办呢？

参考我的邪修大法，直接去现成的ui库、现成的提示词库、现成的网页端，像拿着菜篮子一样，看到喜欢哪个，喜欢怎么拼接就怎么拼接。

指引步骤：
1. 去UI库、开源代码
2. 如果代码没有html原型，先让他根据里面的代码和图片，做出对应的html原型
3. 对比这个html是否符合需求
4. 然后让他按照代码规范根据原型图，添加到哪个页面哪个位置
5. 之后就是微调测试，很多时候都不是一次跑通（codex除外）。

重点查看速查表
1. 看导航栏：logo、菜单、按钮
2. Hero：banner、首屏展示字体、引导
3. Content：图文、卡片、表格
4. Footer：版权、联系信息

这些反复出现的组件，看怎么去对齐留白和分组
- 没有目标-->刷网站库，找审美参考
- 动手开发-->看组件库
- 目标明确-->抄对标网站

提示词速查表
根据原型图，生成对应的代码规范

```
我需要当前业务的整体UI设计做一次风格迭代，我会给你几张参考设计的截图
请你仔细分析它的视觉风格，然后把这种风格应用到我的当前页面上
请重点还原这几个方面：
1. 配色：主色、辅助色、字体颜色、背景颜色
2. 字体：标题和正文的字号层级、粗细、间距、对齐方式、行高
3. 间距与留白： 模块之间的间距、元素之间的呼吸感
4. 组件样式：按钮、卡片、导航栏的圆角、阴影、边框
5. 整体布局：参考它的排版结构来组织页面
```

规范设计文档
```
请把我们这套设计风格保留下来，总结成一份设计规范文档，保存到design.md文件中，内容包括
以后我每做一个新页面，你都要先读这份design.md
1. 配色：主色、辅助色、背景色、、文字色的具体色值
2. 字体：标题和正文的字号、字重、行高
3. 间距：常用的内外边距和模块间距
4. 组件样式：按钮、卡片、导航栏的圆角、阴影、边框规则
严格按照里面的规范来设计，保证整个项目风格统一
```

## 3、设置规范

对于别人的项目你最好别一上来就'/init' 去**初始化操作**，血和泪的教训
1. 技术栈不熟悉：不熟悉你还怎么按照你的想法去构建，全是阻力，像这个博客啊什么是swup啊？？
2. 屎山代码：如果你入手的项目一开始就是屎山，你会越改越难受。
3. 口吐芬芳：”报错你没看到嘛“、”画面怎么黑了“、“不要无视我的需求”、“不用抱歉，帮我改对啊”、“怎么切换页面后css失效了啊”。。。

正确打开姿势
1. 让AI去深度研究这个项目结构、开发注意事项、目录树。指定目标：输出架构图 + 模块依赖 + 核心数据流
2. 重点问：坑在哪、配置怎么搞、调试怎么搞、部署注意啥。要一份 "新人上手清单" 和 "常见翻车现场合集"
3. 你认真踩坑好后，开始/init，然后让他：按照企业级别规范、解耦模块边界等生成代码规范更改claude.md
4. 一定要检查claude.md，比如里面生成提交代码规范的，而你恰好有相关skill，你可以更改成让他使用xxx.skill
5. 最好让他再一次头脑风暴，按照企业级跟自己对齐里面的内容。进行迭代一次


## 4、开发

1. 获取原型html
2. 进行头脑风暴学习原型图，说明需求，对齐需求，生成PLAN
3. 按照代码规范开发（如果不是codex这里需要提醒使用TDDskill相关测试skill，不然后面就是糟心的白屏测试）
4. 各种检查，查交互（无卡顿、断层、闪烁、节流、防抖、重排）、查性能损耗、查代码规范等
5. 确认整体没什么问题，才能提交。

## 5、优化

你别以为上面开发就完了
1. 如果想让自己的页面动起来这里推荐一个提示词保证让你的博客飞起来
```
依次从上往下分层显示。“我想要更像高端设计师作品集/创意机构官网的动效，不要普通淡入。首屏需要有完整openinganimation，标题要有强视觉进场，比如遮罩揭开、位移、压缩后归位。滚动到每个模块时，英文大标题先大幅进场，卡片再依次stagger出现，图片要有 reveal 或轻微parallax。整体动效要夸张一点但高级，节奏慢一些、缓动丝滑，不要廉价弹跳，也不要影响性能。可以用 GSAP +ScrollTrigger。”
```
2. 优化图片、字体一些加载资源，各种压缩，缩小图片，
```
图片：压缩，适当调整分辨率适合对应的位置大小，转换成webp

文字：压缩成日常用字6000字左右，还有包括数字、字母、标点符号和ASCLL码这些字体

懒加载：一些例如瀑布流图片、获取文章数等请求，可以异步加载

还有好多，后续补充...
```
3. 定时AI扫描项目漏洞，代码规范，冗余代码等
